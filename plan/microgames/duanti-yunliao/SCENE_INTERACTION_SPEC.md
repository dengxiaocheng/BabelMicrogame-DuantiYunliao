# SCENE_INTERACTION_SPEC: 断梯运料

## Scene Objects

| 对象 | 渲染方式 | 可交互 | 操作方式 |
|---|---|---|---|
| 脚手架格子 | 52×52 方格，颜色=耐久 | move 阶段可点击相邻格 | 点击 → 移动 |
| 材料卡片 | 矩形卡片，显示名称/重量/价值/稳定 | select_materials 阶段可点选 | 点击 → 选中/取消 |
| 卸货点 | 格子标记（📦），金色边框 | 到达自动触发 | 被动 |
| 断裂格 | 暗色格子 + ✕ 标记 | 不可通行 | 无 |
| 玩家位置 | 格子内👷图标，白色边框+光晕 | 显示当前位置 | 被动 |

## Grid Layout

- 每轮一张模板，尺寸不同（5×5 / 6×6 / 5×7 / 6×9）
- 格子耐久编码：0=缺失，1–5=耐久值
- 起点：蓝色边框标记
- 卸货点：金色边框 + 📦 标记
- 格子颜色映射（durColor）：
  - ≥5: 绿 `#2a9d3a`
  - 4: 深绿 `#4a8d2a`
  - 3: 黄 `#b8a020`
  - 2: 橙 `#d07020`
  - 1: 红 `#d03030`
  - 0: 暗 `#1a1a1a`（断裂）

## Player Input Flow

```
[select_materials] 点击材料卡片 → 选中/取消 → 点击"确认选材"
[plan_route]      查看格子布局 → 点击"确认路线，出发"
[move]            点击相邻格子 → 触发移动结算
[event]           点击事件选项卡片 → 应用效果 → 返回移动
[arrived]         点击"继续搬运" → 回到 select_materials
[accident]        点击"处理事故" → 进入 round_end
[round_end]       点击"进入第N轮" → next_round
[game_over]       点击"重新开始" → restart
```

## Feedback Channels

| 通道 | 渲染方式 | 关联状态 |
|---|---|---|
| 格子耐久颜色 | 方格背景色实时变化 | grid 耐久值 |
| 玩家体力条 | 蓝色进度条 + 数值 | stamina |
| 架体稳定条 | 动态颜色进度条 | scaffoldStability |
| 坠落风险条 | 红/橙/绿三色进度条 | risk |
| 卸货进度条 | 金色进度条 | deliveryProgress |
| 负重数值 | 橙色粗体数字 | loadWeight |
| 回合计数 | N/limit 格式，超 80% 变红 | turn / turnLimit |
| 消息栏 | 左侧金色竖线 + 文字 | message |

## Phase-Specific UI Rules

### select_materials 阶段
- 显示全部 8 种材料卡片，可多选
- 底部显示当前总负重
- 至少选 1 件才能确认

### plan_route 阶段
- 格子网格可观察但不可点击移动
- 显示"确认路线，出发"按钮

### move 阶段
- 仅相邻格子（曼哈顿距离=1）可点击
- 已断裂格子（耐久=0）不可点击
- 点击后即时更新格子颜色和状态条

### event 阶段
- 覆盖层显示事件名称、描述、2 个选项
- 每个选项显示 effects 摘要（资源/压力/风险/关系变化）

## Forbidden UI

- 不允许只用"走近路/走远路"选择
- 不允许做开放地图探索
- 不允许用纯文字按钮列表替代格子点击
- 不允许在首屏无可操作的场景对象

## Acceptance Rule

- 首屏必须让玩家看到至少一个可直接操作的场景对象（材料卡片）
- 玩家操作必须产生即时可见反馈，且反馈能追溯到 Required State
- 不得只靠随机事件文本或普通选择按钮完成主循环
